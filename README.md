### 📊 GitHub Stats

<details>
<summary><b>Private repos & hosting — read this once</b></summary>

Your **apps** can live on Azure, AWS, on-prem, etc. **They do not need to be on Vercel.**  
To count **private** commits in these cards, you only self-host the tiny **[github-readme-stats](https://github.com/anuraghazra/github-readme-stats)** API (Vercel, Railway, Docker, whatever you prefer) and give it a GitHub token.

1. Fork **[anuraghazra/github-readme-stats](https://github.com/anuraghazra/github-readme-stats)** (or use Deploy / import on your host).
2. Deploy the project and add env var **`PAT_1`** = a [Personal Access Token](https://github.com/settings/tokens) with **`repo`** scope (so it can read your private repos you own).
3. Copy your deployment base URL (example: `https://github-readme-stats-xxxx.vercel.app`).
4. In this `README.md`, replace every **`github-readme-stats.vercel.app`** below with that base host (no trailing slash). Commit and push.

The **streak** widget uses a separate service; it may still differ from your full private history unless you also self-host **[github-readme-streak-stats](https://github.com/DenverCoder1/github-readme-streak-stats)** with a token—optional.

</details>
